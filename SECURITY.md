# Security Policy

## Reporting a Vulnerability

Please do **not** open public GitHub issues for security vulnerabilities.

Use GitHub's **private vulnerability reporting** for this repository (Security tab), or contact the maintainers privately.

## Before Publishing

See [docs/SECURITY_SANITIZATION.md](docs/SECURITY_SANITIZATION.md) for a checklist covering secrets, tokens, `.env`, `.mcp.json`, logs, and reports.

## Scope

This is a template/boilerplate repository. The primary security concern is ensuring no real secrets, credentials, or internal URLs are committed. See the sanitization checklist before making your fork public.
